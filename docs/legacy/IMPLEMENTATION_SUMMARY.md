## 🎓 Key Design Decisions

### 1. GUIDs as Strings ✅
- **Why**: LaunchBox stores GUIDs as text in XML
- **Impact**: Direct XML serialization, no conversion needed
- **Benefit**: 100% compatibility

### 2. System.Xml.Serialization ✅
- **Why**: Built into .NET, no external dependencies
- **Impact**: Lightweight, fast, proven technology
- **Benefit**: Easy to maintain, well-documented

### 3. FileSystemWatcher for Cache ✅
- **Why**: Auto-detect changes from LaunchBox
- **Impact**: Always consistent with disk
- **Benefit**: Parallel usage "just works"

### 4. ReaderWriterLockSlim ✅
- **Why**: Multiple concurrent reads, exclusive writes
- **Impact**: Thread-safe cache operations
- **Benefit**: Future-proof for multi-threading

### 5. No Database (XML Only) ✅
- **Why**: 100% LaunchBox compatibility
- **Impact**: Slightly slower than SQL for search
- **Benefit**: Zero migration, instant compatibility
